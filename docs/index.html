what it does

Three parts, one never-idle loop.

01 / QUEUE

One task in flight

Work is a durable on-disk queue. The driver returns exactly three outcomes — wait, launch, or stop — so there is never a fourth 'sit on a timer' state. Idle is impossible by construction.

02 / WAKE

Completion is the cadence

When the in-flight task finishes, that event launches the next unit immediately. A scheduled wake exists only as a long-interval hang-catcher, never as the thing the loop waits on.

03 / CONTINUE

Stop means continue

A reference stop-hook checks the backlog: empty means stop cleanly; work remaining means continue the session. The agent picks itself back up instead of waiting to be told.
